\section{Introduction}
Ce Hardware Maintenance Manual est le document de référence technique pour la maintenance, le démontage, le remplacement de pièces et le débogage matériel du robot Agri-Scout. Il s'adresse à un technicien souhaitant intervenir sur la mécanique ou l'électronique du robot après une panne ou une usure normale.

La majeure partie de la structure mécanique a été conçue et fabriquée par l'équipe : les pièces en plastique sont imprimées en 3D (FDM, PLA) à partir de fichiers CAO Autodesk Inventor, et les supports métalliques sont découpés et percés en laboratoire.

Avant toute intervention : coupez l'interrupteur des batteries LiPo et débranchez la Powerbank du Raspberry Pi.

\section{Système d'Alimentation}

\subsection{Architecture électrique}
Le robot utilise trois sources d'alimentation distinctes et indépendantes pour isoler les circuits sensibles des perturbations des moteurs :

\begin{itemize}
    \item LiPo 5000 mAh (11,1 V) : alimente les moteurs de traction via les ESC.
    \item LiPo 2200 mAh (11,1 V) : alimente le pont en H L298N qui contrôle le moteur pas à pas de la sonde.
    \item Powerbank USB-C : alimente uniquement le Raspberry Pi (5 V), évitant les chutes de tension (brown-outs) lors des pics de courant moteur.
\end{itemize}

Le câble USB entre le Raspberry Pi et l'Arduino a son fil d'alimentation rouge (+5 V) coupé. Seuls le GND et les données sont connectés. La masse commune (GND) entre tous les éléments est impérative pour le bon fonctionnement des communications.

\begin{enumerate}

    \item Insérez la nouvelle batterie et inversez la procédure.
\end{enumerate}

Pour déposer le Raspberry Pi :
\begin{enumerate}
    \item Débranchez la Powerbank USB-C.
    \item Déconnectez le câble USB vers l'Arduino.
    \item Retirez les 4 vis M3 de fixation au châssis.
\end{enumerate}

Pour déposer l'Arduino :
\begin{enumerate}
    \item Notez la connexion de chaque broche avant de déconnecter (ou photographiez).
    \item Débranchez le câble USB vers le Raspberry Pi.
    \item Retirez le câble des ESC (pins 9, 10) et du pont en H (pins 4, 5, 6, 7).
    \item Retirez les vis de fixation.
\end{enumerate}

Lors du remplacement de l'Arduino, il est impératif de re-téléverser le firmware \texttt{main\_firmware.ino} (voir Developer Manual, section Compilation et Modification).

\section{Débogage Matériel}

\subsection{Roues non contrôlées au démarrage}
Symptôme : les roues tournent de façon erratique ou ne répondent pas aux commandes.

Causes possibles :
\begin{itemize}
    \item GND flottant : absence de masse commune entre le Raspberry Pi, l'Arduino et les ESC. Vérifiez la continuité du fil GND du câble USB (entre Pi et Arduino) et la connexion GND entre l'Arduino et chaque ESC.
    \item ESC non initialisé : les ESC RC doivent recevoir un signal neutre (90°) avant de répondre. Le firmware envoie ce signal au démarrage ; si l'Arduino n'est pas encore prêt, les ESC resteront en veille. Redémarrez le système.
    \item Batterie LiPo 5000 mAh insuffisante : une batterie déchargée peut provoquer un comportement imprévisible. Vérifiez la tension avec un voltmètre (minimum 10,5 V pour un pack 3S).
\end{itemize}

\subsection{LIDAR muet ou absent}
Symptôme : le LIDAR tourne physiquement mais le topic \texttt{/scan} est vide ou absent.

Procédure de diagnostic :
\begin{enumerate}
    \item Vérifiez que le port est correctement assigné :
\begin{verbatim}
ls /dev/ttyUSB*
\end{verbatim}
    \item Vérifiez les permissions :
\begin{verbatim}
sudo chmod 666 /dev/ttyUSB0
\end{verbatim}
    \item Relancez le nœud LIDAR et observez les logs :
\begin{verbatim}
ros2 run rplidar_ros rplidar_composition \
  --ros-args -p serial_port:=/dev/ttyUSB0 \
             -p serial_baudrate:=115200
\end{verbatim}
    \item Si le LIDAR est sur \texttt{/dev/ttyUSB1} (conflit avec le capteur sol), échangez les ports USB ou créez des règles \texttt{udev} permanentes.
\end{enumerate}

\subsection{Arduino non détecté}
Symptôme : le nœud Python affiche \texttt{Serial: No device found} et s'arrête.
\begin{enumerate}
    \item Vérifiez la présence du port :
\begin{verbatim}
ls /dev/ttyACM*
\end{verbatim}
    \item Vérifiez les permissions :
\begin{verbatim}
sudo chmod 666 /dev/ttyACM0
\end{verbatim}
    \item Débranchez et rebranchez le câble USB. Le délai de 2 secondes dans le code permet à l'Arduino de finir son reset.
    \item Si le port reste absent, testez l'Arduino sur un autre PC avec l'IDE Arduino pour confirmer qu'il fonctionne.
\end{enumerate}

\subsection{Sonde ne descend pas}
Symptôme : la commande \texttt{D} est envoyée mais aucun mouvement n'est observé.
\begin{itemize}
    \item Vérifiez que la batterie LiPo 2200 mAh est connectée et chargée.
    \item Vérifiez la LED d'état du pont en H L298N : elle doit être allumée.
    \item Vérifiez les connexions des 4 fils du moteur pas à pas sur les sorties A/B du L298N.
    \item Testez le moteur manuellement avec \texttt{calibrate\_probe.py} (voir User Manual).
\end{itemize}

\subsection{Caméra non détectée}
Symptôme : le nœud \texttt{v4l2\_camera\_node} échoue au démarrage.
\begin{enumerate}
    \item Vérifiez la présence du périphérique :
\begin{verbatim}
ls /dev/video*
v4l2-ctl --list-devices
\end{verbatim}
    \item Débranchez et rebranchez la webcam USB.
    \item Si plusieurs caméras sont présentes, précisez le bon périphérique dans le fichier de lancement (\texttt{video\_device: /dev/video0}).
\end{enumerate}

\subsection{Conflit de ports USB (/dev/ttyUSB0 vs /dev/ttyUSB1)}
Le LIDAR et le capteur NPK utilisent tous deux une connexion USB série. Leur assignation (\texttt{ttyUSB0} ou \texttt{ttyUSB1}) peut varier selon l'ordre de branchement. Pour fixer les ports de manière permanente, créez des règles \texttt{udev} basées sur le numéro de série USB de chaque appareil :
\begin{verbatim}
udevadm info -a -n /dev/ttyUSB0 | grep '{serial}'
\end{verbatim}
Puis créez un fichier \texttt{/etc/udev/rules.d/99-agri.rules} avec les règles appropriées.
